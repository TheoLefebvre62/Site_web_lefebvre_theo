SAE 5.03 - Déployer et gérer des services ROM avec Asterisk et WebRTC

Introduction au projet

L’objectif de cette SAE était de concevoir, déployer et sécuriser une solution de communication multimédia dans un environnement réseau opérateur. Le projet portait sur l’intégration de services VoIP et de visioconférence via un serveur Asterisk, et leur administration à travers une interface web personnalisée. Les technologies clés employées sont Asterisk, SIP.js, WebRTC, Alpine Linux et des serveurs web (Apache/NGINX).

Les enjeux principaux étaient :

  • Mettre en place un serveur Asterisk fonctionnel sur une machine virtuelle.
  • Développer un portail web pour la communication voix, vidéo et messagerie instantanée.
  • Assurer la sécurité des échanges via des protocoles comme TLS.
Description technique détaillée
  • Déploiement du serveur Asterisk
    • Machine virtuelle sous Alpine Linux déployée sur un réseau VLAN 800.
    • Installation manuelle d’Asterisk, configuration des services, vérification du bon fonctionnement.
  • Connexion des téléphones IP
    • Configuration et enregistrement de deux téléphones (FANVIL et CISCO).
    • Mise en place d’un plan de numérotation.
    • Démonstration de la communication voix entre terminaux, analyse avec Wireshark.
  • Accès à l’administration d’Asterisk
    • Déploiement d’un serveur HTTP intégré à Asterisk (sans solutions préconfigurées comme FreePBX).
    • Création d’une interface web minimale pour accéder à l’administration du serveur.
  • Déploiement d’un serveur web
    • Serveur web (Apache ou NGINX) installé sur Alpine Linux.
    • Création d’une page d’accueil et vérification de l’accessibilité.
  • Intégration SIP.js pour la voix via WebRTC
    • Étude du protocole WebRTC.
    • Implémentation d’une communication voix à l’aide de SIP.js.
    • Traçage et analyse des paquets échangés via Wireshark.
  • Extension de l’interface : vidéo et messagerie instantanée
    • Ajout des fonctionnalités vidéo et chat à l’interface web.
    • Développement d’un portail complet de visioconférence.
  • Audit de sécurité
    • Réalisation d’un audit réseau (analyse MITM, sniffing).
    • Observation des faiblesses et des échanges non sécurisés.
  • Sécurisation des échanges
    • Implémentation du protocole TLS pour chiffrer les flux.
    • Validation du chiffrement par analyse réseau.
Compétences développées
  • Virtualisation et administration système : Déploiement de VM, configuration réseau.
  • Téléphonie IP et multimédia : Installation et configuration d’Asterisk, SIP, WebRTC.
  • Développement web : Utilisation de SIP.js, création d’interfaces pour la communication multimédia.
  • Cybersécurité : Audit, analyse de paquets, mise en œuvre de TLS.
  • Travail en équipe et gestion de projet : Organisation en trinômes, progression par étapes validées.
Réflexion personnelle

Ce projet a offert une approche complète de la gestion de services multimédias dans un cadre opérateur. Il a permis d’appliquer des compétences techniques pointues en virtualisation, VoIP, développement web et cybersécurité. La réalisation d’un portail de visioconférence personnalisé et sécurisé constitue une application concrète des technologies vues en cours. Cette SAE a renforcé notre capacité à concevoir des solutions complètes, depuis le déploiement jusqu’à la sécurisation.


SAE 5.01 - Concevoir, réaliser et présenter une solution technique : Infrastructure virtualisée avec Proxmox et accès distant sécurisé

Introduction au projet

Dans le cadre de la SAE 5.01, nous avons conçu, déployé et documenté une solution complète d’infrastructure virtualisée à l’aide de Proxmox VE, avec un accès distant sécurisé via Apache Guacamole. Ce projet visait à mettre en place un environnement permettant la gestion et l’utilisation de machines virtuelles pour des travaux pratiques dans un cadre pédagogique.

L’objectif principal était double :

  • Créer une plateforme de virtualisation centralisée, accessible et performante.
  • Assurer un accès distant sécurisé aux VM à l’aide d’une interface web, tout en gérant les droits des utilisateurs.
Description technique détaillée
  • Choix de la solution technique
    • Comparaison détaillée entre plusieurs hyperviseurs (Proxmox, VMware, Hyper-V, Xen).
    • Choix de Proxmox VE pour sa gratuité, son interface web intuitive, la gestion intégrée des conteneurs (LXC), le support de la haute disponibilité, et une communauté active.
  • Installation et configuration de l’hyperviseur
    • Création d’une machine virtuelle Proxmox avec double interface réseau (bridge + NAT).
    • Attribution d’une IP statique et mise en place d’un tunnel VPN WireGuard pour un accès distant sécurisé à l’interface Proxmox.
    • Définition des ressources : 8 cœurs CPU, 16 Go RAM répartis entre Proxmox, VMs Ubuntu Desktop et Debian.
  • Création des machines virtuelles modèles
    • Déploiement de 3 types de VM : Ubuntu (avec interface graphique), Debian (sans interface graphique) et Alpine.
    • Préparation des modèles avec installation des services nécessaires (x11vnc, SSH, Wireshark).
    • Conversion des VM en modèles clonables pour les futurs utilisateurs.
  • Déploiement de l’interface d’accès distant avec Apache Guacamole
    • Installation manuelle de Guacamole Server et de son client web sur une machine Ubuntu.
    • Intégration avec MariaDB pour l’authentification centralisée.
    • Configuration du serveur Tomcat et déploiement de l’application web.
    • Connexions possibles via SSH pour Debian et VNC pour Ubuntu, configurées dans Guacamole.
  • Gestion des utilisateurs et des droits
    • Création des utilisateurs et d’un groupe “Élève” sur Proxmox, avec droits limités pour cloner et utiliser les VM.
    • Configuration identique sur Guacamole pour restreindre les connexions selon les rôles.
  • Documentation et durabilité
    • Rédaction de manuels d’utilisation pour Proxmox et Guacamole.
    • Réflexion sur les enjeux de développement durable : réduction des déplacements, mutualisation des ressources matérielles, diminution de la consommation énergétique.
Compétences développées
  • Virtualisation : Maîtrise de Proxmox, création de modèles de VM, gestion des ressources.
  • Réseaux et sécurité : Mise en œuvre d’un VPN WireGuard, configuration d’accès sécurisé distant.
  • Systèmes et administration : Déploiement de services sur Linux (Alpine, Debian, Ubuntu).
  • Développement et intégration de services : Installation complète d’Apache Guacamole avec base de données.
  • Gestion de projet et documentation : Matrice RACI, tutoriels détaillés, manuels d’accès.
Réflexion personnelle

Ce projet nous a permis de mettre en œuvre une infrastructure complète, professionnelle et sécurisée, avec des outils open-source puissants comme Proxmox et Guacamole. La gestion fine des accès et la préparation d’environnements prêts à l’emploi constituent des apports concrets pour des cas d’usage en entreprise ou en formation. La dimension durable et l’optimisation des ressources soulignent l’importance d’intégrer des pratiques responsables dans les projets techniques.


SAE5.Cyber.03 - Supervision de la sécurité avec la suite Elastic

Introduction au projet

Dans le cadre de la SAE5.Cyber.03, nous avons été chargés de mettre en place une solution de supervision de la sécurité informatique à l’aide de la suite Elastic (ELK Stack) sur un environnement Ubuntu. L’objectif principal était de centraliser, visualiser et analyser les logs de différents systèmes (machines, services et équipements) dans une démarche de détection proactive des incidents de sécurité.

Ce projet est également prolongé dans le cadre du module R5.Cyber.11, à travers des exercices d’analyse de logs (TP0 à TP5) pour mettre en pratique l’exploitation de la supervision dans un contexte réaliste.

1. Installation de la suite Elastic sur Ubuntu
  • Distribution utilisée : Ubuntu 22.04 LTS
  • Composants installés :
    • Elasticsearch
    • Logstash
    • Kibana
    • Filebeat / Packetbeat (selon les sources de logs)
  • Méthodologie :
    • Ajout des dépôts Elastic et import de la clé GPG
    • Installation via apt des composants nécessaires
    • Configuration de elasticsearch.yml pour permettre l’accès externe
    • Paramétrage de kibana.yml avec les bonnes adresses et ports
    • Configuration de sécurité basique : utilisateurs, HTTPS, authentification
  • 🔧 Validation via l’accès à l’interface Kibana sur le port 5601
2. Supervision d’une machine Ubuntu
  • Agent utilisé : Filebeat
  • Logs collectés :
    • Authentification (/var/log/auth.log)
    • Système (/var/log/syslog)
    • Réseau (optionnel via Packetbeat)
  • Tableaux de bord :
    • Visualisation des connexions SSH
    • Tentatives de connexion échouées
    • Création de comptes utilisateurs
  • Résultat : Détection d’un scan de ports et de plusieurs tentatives de connexion brute-force
3. Supervision d’un service web (Apache/Nginx)
  • Service surveillé : Apache2 (/var/log/apache2/access.log)
  • Agent utilisé : Filebeat avec module apache
  • Dashboards Kibana :
    • Activité HTTP par heure
    • Statistiques de codes HTTP
    • Origine des requêtes (géolocalisation via GeoIP)
  • Résultat :
    • Identification de scans de vulnérabilités (ex: requêtes vers /phpmyadmin)
    • Détection de pics d’activité suspects à des heures creuses
4. Supervision d’équipements Cisco
  • Équipements utilisés : Routeur/Switch Cisco
  • Méthode :
    • Activation du service Syslog sur l’équipement
    • Redirection des logs vers le serveur Ubuntu via UDP
    • Ingestion dans Logstash, parsing personnalisé
  • Logs supervisés :
    • Connexions, déconnexions, changements de configuration
    • Logs ACL
  • Résultat : Détection de rechargements manuels et de modifications suspectes de configuration
5. Supervision d’un autre équipement/service
  • Service supervisé : OpenVPN (ou fail2ban, firewall UFW…)
  • Méthode :
    • Configuration de Filebeat pour surveiller /var/log/openvpn.log (ou ufw.log)
    • Création de visualisations : nombre de connexions VPN, blocages IP, etc.
  • Résultat :
    • Détection de tentatives de connexions étrangères non autorisées
    • Analyse des tendances d’utilisation de VPN

R5.Cyber.11 - Analyse de logs (TP0 à TP5)

  • TP0 : Découverte des formats de logs et structures JSON
    • Identification de loglines critiques
    • Transformation via Grok dans Logstash
  • TP1 : Analyse de logs de connexions SSH
    • Identification de brute-force
    • Corrélation par adresse IP
  • TP2 : Logs Apache – détection d’activités malveillantes
    • Analyse de requêtes anormales
    • Création d’un filtre sur les URI sensibles
  • TP3 : Équipement réseau
    • Parsing des logs Cisco
    • Extraction des messages d’alerte
  • TP4 : Supervision temps réel avec alertes
    • Implémentation d’alertes dans Kibana ou via Watcher
    • Cas d’usage : tentative d’intrusion, dépassement seuils
  • TP5 : Corrélation multi-sources
    • Fusion de logs Apache, SSH et VPN
    • Détection de comportements anormaux combinés
Compétences développées
  • Déploiement d’outils SIEM : ELK Stack, agents Filebeat et Logstash
  • Collecte & parsing de logs : logs système, web, réseau, équipements Cisco
  • Visualisation & alerte : création de dashboards, détection d’anomalies
  • Sécurité opérationnelle : détection d’intrusions, analyse forensique
  • Méthodologie d’analyse : structuration, corrélation, investigation multi-sources
Réflexion personnelle

Ce projet nous a permis de développer une solution complète de supervision basée sur des outils professionnels, en intégrant des problématiques réelles de sécurité informatique. La richesse des logs collectés et la puissance de la suite ELK nous ont permis d’apprendre à détecter des menaces, investiguer des incidents, et structurer nos analyses de manière pertinente. Ce projet constitue une base solide pour des missions futures dans un SOC (Security Operations Center) ou en audit sécurité.
